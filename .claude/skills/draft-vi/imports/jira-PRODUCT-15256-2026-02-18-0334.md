## Why

The existing React Grid library is no longer actively maintained and is technically outdated.

A thorough explanation and assessment is provided by Kibana, which was also used and migrated off React-Grid, here:
https://www.elastic.co/search-labs/blog/kibana-dashboard-build-layout

## What

Should offer at least the same or better experience (UX/performance) than provided by React Grid today. Key characteristics for React grid can be split into **must-haves** (essential for functionality and usability) and **nice-to-haves** (features that enhance the experience but aren't critical):

### Must-Haves to Be On Par with the Old Library (Enablement)

#### 1. Core Features

* **Drag-and-Drop:** Smooth, intuitive drag-and-drop with constraints (grid-based).
* **Responsive Design:** Breakpoint support for device-specific layouts and automatic resizing as currently.
* **Dynamic Resizing:** Resizable elements with real-time updates.
* **Grid System:** Configurable grids with snapping, row/column alignment, and gutters.
* **Additional layout support:** Support for other layouts, for example, canvas layout.
* **Multi-selection:** Ability to still multi-select tiles and move them.

#### 2. Performance

* **Optimized Rendering:** minimal re-renders.
* **Lightweight:** Small bundle size and efficient updates.
* **Batch Operations/Multi-selection:** Efficient processing of multiple layout changes in one go

#### 3. Developer Experience, Accessibility, and Maintenance

* **Easy Integration:** Clear APIs, TypeScript support, and documentation.
* **Event Handling:** Callbacks for drag, resize, and layout changes.
* **State Management:** Controlled layouts with external state options.
* **Error Handling:** Debugging knowledge in the team and graceful degradation for invalid configurations.
* **Keyboard Navigation:** Full keyboard support for layout interactions.
* **Screen Reader Support:** WAI-ARIA compliance and clear labels for accessibility.
* **Cross-Browser Support:** Compatibility with all major browsers.
* **Touchscreen support:** works on mobile devices

### New Functionality to Better Manage Layouting/Layouts

1. duplicating multiple tiles - logic - mirror tiles on the horizontal bottom axis - this means below the respective original one (see illustration for the left one the upper one is correct, the lower one would be option 2 which we skipped…)
2. expanding multiple tiles
   * tiles **not** horizontally overlapping can be expanded in height
   * tiles **not** vertically overlapping can be expanded in width

### What Is It Not?

1. layout options: distribute (same size of all tiles…)
2. ~~grouping~~ => descoped (add ideas/options to new VI PRODUCT-5653)
