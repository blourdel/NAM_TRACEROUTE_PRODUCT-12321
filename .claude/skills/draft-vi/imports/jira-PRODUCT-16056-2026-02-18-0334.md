# PRODUCT-16056 - Migrate AppShell app from Hermes

**Status:** Usecases defined
**Type:** Technical enabler (simple)
**Assignee:** Simon Ludwig
**Execution Assignee:** Damian Skrodzki
**Owning Program:** Platform Apps
**Created:** 2026-01-15
**Updated:** 2026-02-09

## Summary

AppShell currently uses an outdated deployment method for "built-in" apps – Hermes, and it needs to be migrated.

**Option 1️⃣**: AppShell becomes a Dynatrace App and is releases as one. (⭐ **chosen variant**)
**Option 2️⃣**: AppShell service (Platform Search service) could be extended to cover some backend for frontend use cases, including serving the AppShell app.

Decision notes: https://miro.com/app/board/uXjVPkjvAJo=/?share_link_id=421082667982

## Proposed Solution

We want to migrate AppShell to be deployed as every other app – through Hub Console. This way we won't change the "interface" between AppShell and external components (AppRegistry, Hub) so few external changes are required. The major change is how the team will release the app and promote through the stages.

## Acceptance Criteria

1. AppShell becomes a pre-install app.
2. AppShell no longer publishes Helm Charts
3. The team updates agreed release schedule to
   - Manually or automatically release AppShell every sprint
   - Attach sprint version to the AppShell version
   - Release a backport to any dev/hard/production version

## Not in Scope

1. Removing the need to use Chrome Extension to develop AppShell
2. Implementing built PR preview experience for AppShell
