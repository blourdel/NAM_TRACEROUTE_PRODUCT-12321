## Short Abstract

As addon intents become more widely adopted, the demand for more significant flexibility increases. Users request additional sizes for the addon modal (already delivered), support for sheets, and various design optimizations. This VI aims to implement these enhancements to boost the overall adoption of addon intents while ensuring a delightful user experience.

## What It Is

* **Support sheets**
  * Add a new bottom sheet view in addition to the current views.
  * https://developer.dynatrace.com/design/components-preview/overlays/Sheet/
* **Modal anatomy**
  * Provide an open variant that allows teams to freely change:
    * modal width and height
      * Width: one of Small/Medium/Large?
      * Height: fixed based on the size or automatically adjusted is optionally adjusted automatically using a new API
    * background
    * header
* **Navigation**
  * ~~Stack addons instead of closing them.~~
    * Check if we can apply any workaround or have a dedicated VI for this
  * ~~Handle toast gracefully → We'll be suggesting an alternative for system feedback. Toast should not be used within addons.~~
* **Improve documentation**
  * Update developer documentation
    * Add usage examples - best practices
  * Update customer-facing documentation
  * Create design documentation
    * Explain standard anatomy - How to avoid bg color issues, how to deal with headers etc.
    * Showcase standard addon examples - e.g. creation flow
    * Describe how to avoid stacking (modal in modal) and how to handle system feedback (without toasts)
